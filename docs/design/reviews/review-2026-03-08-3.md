### 1. Bayesian utility score computation — Correct

`artifact_store.py:851-863` implements the Bayesian average exactly as specified:

```python
utility_score = (count_useful + 1) / (count_total + 2)
```

The COUNT query uses `FILTER (WHERE useful = TRUE)` to get `count_useful`, which is correct. The Laplace smoothing prior (+1/+2) matches the spec. Cold start at 0.5 with no feedback is correct.

One observation: the existence check, version lookup, INSERT, COUNT, and UPDATE in `record_feedback` are all separate queries within a single `pool.acquire()` but **not within a transaction**. If two agents concurrently submit feedback for the same artifact, the second agent's COUNT could see stale data, producing a slightly wrong utility score. The score self-corrects on next feedback, and concurrent same-artifact feedback is unlikely in a single-user system, so this is **low severity**. Wrapping in `async with conn.transaction()` would eliminate the race.

### 2. Search CTE scoring — Correct, matches spec

`artifact_store.py:429-461` implements the CTE with all four scoring components:

| Component | Spec | Implementation | Match? |
|-----------|------|----------------|--------|
| base_score | `1 - (ae.embedding <=> $1::vector)` | Same | Yes |
| confidence_boost | HIGH=+0.1, MEDIUM=0, LOW=-0.1, SUPERSEDED=-0.2 | Same, plus `ELSE 0.0` for NULL safety | Yes |
